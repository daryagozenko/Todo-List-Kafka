## 1. Перед запуском нужно создать .env в корне проекта
и заполнить все перемнные своими данными, то есть все переменные из docker-compose

### 2. Настройки
все настройки БД берутся из переменных окружения, которые я передаю через docker-compose, 
и кладутся в файл application.properties

## 3. Сборка .jar файлов
Так как я решила организовать родительский pom.xml, чтобы не дублировать зависимости,
возникли сложности с докерфайлами DataMS и ApiMS.
context: . - конкретно эта строчка docker-compose не позволяла видеть jar файл,
собираемый maven по настройкам докерфайла. Решила оставить в докерфайлах только запуск
а сами jar генерировать через терминал maven руками:

- mvn clean package -DskipTests -pl ApiMS
- mvn clean package -DskipTests -pl DataMS

Можно было бы отказаться от родительского pom файла и оставить докерфайл как
из 1-ой лабораторной (с этапом билда), но в этой работе решила оставить так

## 4. Запуск
Запускать в терминале: docker-compose up -d

## 5. Тестирование
- http://localhost:8080/swagger-ui/index.html

## 6. О настройках Kafka
KAFKA_BROKER_ID: 1 - уникальный идентификатор узла
KAFKA_ZOOKEEPER_CONNECT: zookeeper:2181 - адрес Zookeeper
KAFKA_ADVERTISED_LISTENERS: PLAINTEXT://kafka:9093 - адрес, который клиент 
использует для подключения внутри docker сети
KAFKA_OFFSETS_TOPIC_REPLICATION_FACTOR: 1 - количество копий данных для одного узла

- Healthcheck этап проверяет, что Kafka готова к работе через команду kafka-topics --list

SPRING_KAFKA_BOOTSTRAP_SERVERS=kafka:9093 - куда подключаться к Kafka другим микросервисам

- Топики создаются автоматически когда сервис отправляет или подписывается на сообщение.
Но я еще дополнительно создала класс KafkaTopicsConfig в ApiMS, чтобы было централизованное и 
наглядное создание топиков, плюсом через конфиг как раз таки регулируется количество партиций
в топике и прочее.

Producer отправляет сообщение в топик - у меня это ApiMS
Consumer читает сообщение из топика - это DataMS

### 7. RestClient
- По заданию post запросы отправляются от одно МС к другому через кафку
- Get запросы получаются от DataMS напрямую через RestClient
- В ApiMS в application.properties конфигурация для доступа к api DataMS внутри docker сети
  - app.gozenko.url=http://data-service:8080
